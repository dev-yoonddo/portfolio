1. 게시판 데이터 구조 개선
🔴 문제 상황

기존 게시판은 게시물 하나를 작성하면 게시판 항목(FIELD) 수만큼 행이 생성되는 구조였습니다. 항목이 많을수록 게시물 1건 조회에 읽어야 할 행 수가 선형적으로 증가했고, 데이터가 쌓일수록 성능이 저하되는 구조적 한계가 있었습니다.

게시물 1건 조회 시 → 10행 스캔 게시물 100건 조회 시 → 1,000행 스캔 게시물 1,000건 조회 시 → 10,000행 스캔 (데이터 증가에 비례해 성능 저하)
🟢 해결 방법

TB_POST_FIELD를 데이터 타입별 4개 테이블로 분리하고, 컬럼 매핑 정보를 기반으로 어떤 타입 테이블의 몇 번 컬럼을 읽을지 동적으로 결정하는 조회 로직을 구현했습니다.

테이블담당 타입컬럼 구성
TB_POST_TEXT문자열TEXT_01 ~ TEXT_50
TB_POST_NUM숫자NUM_01 ~ NUM_50
TB_POST_DATE날짜DATE_01 ~ DATE_50
TB_POST_CLOB대용량 텍스트CLOB_01 ~ CLOB_50
게시물 1건 조회 시 → 타입별 테이블 최대 4행 고정 게시물 1,000건 조회 시 → 최대 4행 고정 (항목 수 무관)
2. 프론트엔드 동적 렌더링 개선

TB_BOARD_FIELD에 저장된 항목 정의를 백엔드에서 내려주면, 프론트에서 이를 읽어 항목 유형에 맞는 UI 요소를 동적으로 생성하는 구조로 전환했습니다. 기존에는 게시판 항목이 추가되거나 변경될 때마다 개발자가 직접 HTML을 수정해야 했습니다.

① 게시판 진입 시 항목 정의 조회 (TB_BOARD_FIELD)

② 백엔드에서 항목 목록 + 타입 정보 응답

③ 프론트에서 타입별 UI 요소 동적 생성

   ├─ 작성자 → text input

   ├─ 작성일자 → date picker

   ├─ 내용 → textarea

   └─ 파일첨부 → file upload

④ 작성 완료 시 컬럼 매핑 기반으로 타입별 테이블에 저장

3. 평가 데이터 조회 성능 개선
🔴 문제 상황

평가 데이터 조회 시 평균 응답 시간이 40초 이상 소요되었습니다. 실행계획 및 로그 분석을 통해 N+1 쿼리 구조를 확인했습니다.

평가 목록 조회 (1회) └─ 각 평가 항목마다 상세 데이터 조회 (N회 반복) → 항목 100개 기준: 쿼리 101회 발생
🟢 해결 방법
  • N+1 구조 제거: 상세 데이터를 리스트로 일괄 조회 후, Java 반복문으로 매핑하는 방식으로 재설계
  • 인덱스 적용: 조회 조건으로 빈번하게 사용되는 컬럼에 인덱스 추가
  • 쿼리 구조 개선: 불필요한 서브쿼리 제거 및 조회 범위 최소화
평가 목록 조회 (1회) 평가 상세 전체 일괄 조회 (1회) └─ Java 반복문으로 매핑 처리 → 항목 수와 무관하게 쿼리 2회 고정

Spec 기반 UI 렌더링 공통 프레임워크 (진행 중)
🔴 배경 및 문제

각 관리 메뉴마다 HTML + JS 파일이 1:1로 존재하는 구조였습니다. 메뉴가 추가될 때마다 동일한 패턴의 코드를 반복 작성해야 했고, UI 변경 시 각 파일을 개별적으로 수정해야 하는 유지보수 부담이 있었습니다.

사용자관리.html + 사용자관리.js → UI 직접 작성 메뉴관리.html + 메뉴관리.js → UI 직접 작성 코드관리.html + 코드관리.js → UI 직접 작성
🟢 해결 방법
공통 렌더러 (renderView, renderFields, renderModal ...) 사용자관리.js → pageSpec 선언만으로 화면 자동 생성 메뉴관리.js → pageSpec 선언만으로 화면 자동 생성 코드관리.js → pageSpec 선언만으로 화면 자동 생성
  • fieldRenderer 맵핑 테이블: text, select, radio, checkbox, tree, datetimepicker 등 17개 타입 렌더링 함수 분리
  • 렌더링 파이프라인 계층 설계: renderView → renderFields → renderRow → getRenderer 순서로 위임
  • 모달 Spec 기반 구조: 모달 ID, 필드 구성, 푸터 버튼까지 Spec으로 선언, renderModal로 자동 생성
  • actions 구조: Ajax 호출 + 렌더링을 Spec 안에 선언 → onOpen / click / keydown 시점별 자동 실행
  • $.initModalById("groupModal") 한 줄로 렌더 + 데이터 로드 + 이벤트 바인딩 완료
